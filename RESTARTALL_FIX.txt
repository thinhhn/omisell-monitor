RESTARTALL REDIRECT FIX
================================================================================

ISSUE: Click "Restart All" button shows raw HTML instead of redirecting

ROOT CAUSE:
  1. base_url() returns empty string (config not set)
  2. redirect(base_url()) redirects to '/auth' instead of home
  3. Browser displays HTML response instead of following redirect

SOLUTION:
  1. Auto-detect base_url in config.php
  2. Change redirect(base_url()) to redirect('welcome')
  3. Increase sleep time from 2s to 5s for stopAll/startAll

FILES MODIFIED:
  • application/config/config.php
    - Added auto-detection for base_url
  
  • application/controllers/control.php
    - restartall() function: redirect('welcome')
    - Increased sleep: 2s → 5s

DEPLOYMENT:
  scp application/config/config.php user@server:/var/www/supervisord-monitor/application/config/
  scp application/controllers/control.php user@server:/var/www/supervisord-monitor/application/controllers/

TESTING:
  1. Go to: https://supervisord.omisell.com/
  2. Click "Restart All" on any server
  3. Should redirect to homepage with success message
  4. No raw HTML should be displayed

EXPECTED RESULT:
  ✅ Redirects to homepage correctly
  ✅ Shows success message
  ✅ All processes restart successfully
  ✅ No raw HTML displayed
